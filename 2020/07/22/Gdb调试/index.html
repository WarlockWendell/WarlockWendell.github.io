本篇博文记录一些GDB的基本使用方法

在使用GDB调试前，首先要生成可执行程序

使用-g参数，使生成的可执行程序中包含调试信息

然后使用dgb app进入调试截面

1. 打断点

  • 查看程序源代码

    上面的代码查看带main函数的文件

    上面的代码查看a.c函数的第20行内容，行号也可以换成函数名称

    继续使用l就能继续往下看

  • 断点

    使用break n在第n行设置断点，这个break可以简写成b

  • 带有条件的断点

    比如在循环里面，不希望每次都断点

    在i==10时，才会有这个断点

  • 使用info b查看断点信息，info可以简写为i

  • d 编号删除断点

2. 执行

  • start执行一步
  • n单步执行
  • s单步进入
  • c继续执行
  • p i查看i的值
  • ptype i查看i类型
  • display i 追踪变量i的值，使用undisplay 编号 不追踪，编号使用info display查看
  • 进入循环，使用u跳出单次循环
  • finish跳出函数
  • set var i=10将i设置成10
